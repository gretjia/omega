# 训练数据集特性分析报告 (Training Data Analysis Report)

**生成日期**: 2026-01-31  
**项目**: OMEGA Quant System  
**分析对象**: `./data/history_ticks_full` (全市场筛选片段库)

---

## 1. 概览 (Overview)

该数据集包含 **24,015** 个独立的 CSV 数据片段，涵盖了全市场 4000+ 股票在特定时间窗口内的 Tick 级行情。经过对 200 个随机样本的深度统计分析，我们发现该数据集并非随机切片，而是具有显著的 **"高波动"** 与 **"跨日延续"** 特征，非常适合用于训练捕捉中短线趋势与应对极端行情的策略模型。

---

## 2. 核心特性洞察 (Key Insights)

### 2.1 跨日片段为主 (Multi-day Continuity)
绝大多数数据文件实际上包含了 **连续 2 天或以上** 的行情，而非单日数据。这表明数据集旨在训练模型处理 **隔夜风险 (Overnight Risk)** 和 **跨日趋势 (Inter-day Trend)**。

*   **跨度分布**:
    *   **2天**: 占比 **80%** (主流格式)
    *   **4天及以上**: 占比 **18%**
    *   **单日**: 仅占 **2%**
*   **平均时长**: 约 **3,032 分钟** (含休市时间)，覆盖完整的 T+1 交易周期。

### 2.2 显著的高波动性 (High Volatility)
数据集表现出明显的筛选痕迹，倾向于保留价格波动剧烈的时段。

*   **平均振幅**: **5.24%** (显著高于 A 股日均 2-3% 的水平)
*   **极端行情**: 样本中最大振幅达 **21.46%**，且包含 **-11.2%** 至 **+14.5%** 的涨跌幅区间。
*   **训练价值**: 这种分布有利于增强模型对 **突破 (Breakout)**、**反转 (Reversal)** 及 **涨跌停板** 附近的博弈能力的泛化性。

### 2.3 数据粒度 (Granularity)
*   **Tick 密度**: 平均每个文件包含 **7,657** 个 Tick。
*   **时间间隔**: 平均间隔约 **3秒** (符合 Level-1 Snapshot 标准)，但在清淡时段或停牌期间会有拉长。
*   **时间范围**: 覆盖 **2025年1月 - 2026年1月** (截止至当前系统日期的最新历史数据)。

## 4. 数据格式说明 (Data Format)

文件夹中主要包含两种 CSV 格式，训练前需注意 **格式对齐 (Normalization)**：

1.  **RQ 标准格式 (Standard)**:
    *   包含完整字段: `datetime`, `open`, `high`, `low`, `last`, `volume`, `total_turnover`, 及五档盘口 (`a1-a5`, `b1-b5`)。
    *   时间格式: `YYYY-MM-DD HH:MM:SS`

2.  **简化格式 (Simplified)**:
    *   仅包含核心字段: `time` (毫秒时间戳), `price`, `vol`, `ask1`, `bid1`。
    *   **注意**: 部分老旧或特定来源的数据采用此格式，缺少五档深度。

## 5. 建议 (Recommendations)

*   **策略适用性**: 强烈推荐用于 **Swing Trading (波段交易)**、**Event-Driven (事件驱动)** 及 **Volatility Arbitrage (波动率套利)** 策略的训练。
*   **预处理需求**: 建议在训练 Pipeline 中增加一步 **格式标准化**，将所有简化格式数据填充或转换为标准 RQ 格式，特别是统一时间戳格式。
*   **过拟合风险**: 由于数据集中高波动样本占比较高，需注意模型在低波动震荡市 (Choppy Market) 中的表现，建议补充部分低波动样本以平衡训练集。
