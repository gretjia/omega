# OMEGA 训练数据需求（vNext）

本文汇总当前代码库与审计结论，给出 OMEGA 训练数据的**明确需求**与**可执行约束**。

## 1. 最小可运行字段（必须）
- time: 可解析为绝对时间戳（建议 epoch ms），可恢复交易日与顺序
- price: 成交价或最新价，非负且可用于 log-return
- volume: 单笔或快照成交量（若为累积量必须可稳定转换为 delta）

## 2. 强烈建议字段（提升结构识别与边界保护）
- ask1 / bid1: L1 最优报价（用于 OFI/方向代理）
- pre_close / limit_up / limit_down: 或可推导涨跌停价的字段
- turnover/amount, trading_date: 便于校验与日历切分
- 可选更高密度: a1-a5 / b1-b5 / l2transaction / l2order

## 3. 时间跨度与市场状态覆盖
- 至少 2-3 年覆盖（建议 2023-2026）
- 覆盖不同 regime: 震荡、趋势、极端波动、政策冲击
- 不仅高波动样本，还需要低波动/均衡样本

## 4. 数据集结构（两阶段并行）
- Stage 1（结构预训练）: 事件驱动切片（静默→爆发），多日连续、包含隔夜
- Stage 2（泛化微调）: 连续全年 tick（按股票/年份整段）

## 5. 事件切片规范（建议模板）
- 事件日 t0: 量能放大 + 振幅扩张 + 突破区间（阈值用分位数）
- 静默期: t0-20 至 t0-3 低波动/低量分位
- 切片窗口: [-2,-1,0,+1,+2] 为基准，可扩展到 [-5,+5]
- t0 必须置中，避免“起始日跨多日不定长”的混乱

## 6. Volume Clock 可行性约束
- 单文件需足够 tick 生成 volume bars
- 每个样本必须满足:
  - lookback_bars + label_horizon_bars（默认 128 + 16）
  - embedding span（(dimension-1)*delay）条件

## 7. 质量与清洗约束（硬性）
- 时间严格单调；去重；无乱序
- 过滤 price <= 0 且 volume == 0 的非交易段
- 负成交量、异常值剔除
- 编码可自动识别（UTF-8/GBK 等）
- 除权除息日需标记或剔除

## 8. 切分与泄漏控制
- 切片库: 按 (stock, date_block) 分组切分，避免多日重叠泄漏
- 连续库: 优先时间切分（walk-forward）

## 9. 覆盖与均衡约束
- 行业/市值/流动性分层覆盖
- 覆盖吸筹/派发/震荡多种结构
- 涨停/跌停附近样本需保留但标记边界状态

## 10. 落盘与组织规范
- 切片文件: {code}_{YYYYMMDD}.csv
- 连续文件: {code}.csv
- 统一字段命名与单位
- 输出 manifest: 文件 -> 时间范围 -> 字段 -> 校验信息

---
结论: 数据必须既满足 “结构可学习”（Epiplexity / TDA / SRL 所需的微结构密度），也满足 “可泛化”（跨年跨 regime 覆盖与严格切分）。
