**1. 项目定位与三位一体 (Trinity)**
OMEGA vNext 是 A 股量化内核，核心哲学是三位一体：
1. TDA（拓扑几何）：用 Takens 嵌入 + 持久同调刻画“结构形状”。
2. SRL（平方根法则）：冲击物理模型，残差用于冰山流动性探测。
3. Epiplexity & Time‑Bounded Entropy：基于 MDL 的结构/噪声分解，决定可交易性。

关键结论：三者必须同时参与决策，且在“Volume Clock（等量时钟）”上运算。

---

**2. Constitution 的不可违背原则**
来自 `OMEGA_CONSTITUTION.md` 与 `.agent/skills/omega_development/SKILL.md`：
1. **禁止静态交易参数**：阈值必须是动态分布函数（如 rolling quantile），不能硬编码。
2. **Volume Clock**：所有计算必须基于等量 K 线，而非物理时间。
3. **递归闭环**：Epiplexity 调整 TDA 分辨率，Entropy 控制仓位衰减。
4. **边界条件**：接近涨跌停时 SRL 失效，必须禁用或切换边界逻辑。
5. **T+1 约束**：回测/实盘必须遵守结算规则。
6. **冻结快照**：训练结果写入 `artifacts/*.pkl`，严禁回写 `config.py`。

**3. 核心代码结构与职责（重要更正）**
当前 v3 **核心代码在 `omega_v3_core/`，不在根目录**。根目录版本主要作为 L1/L0 参考或旧版保留。

v3 主干（以 `omega_v3_core/` 为准）：
1. `omega_v3_core/kernel.py`：L2 推理核心（递归闭环 + 信号综合 + 仓位控制）。
2. `omega_v3_core/omega_math_core.py`：数学内核（MDL/物理诊断/拓扑面积等 v3 逻辑）。
3. `omega_v3_core/omega_etl.py`：L2 数据 ETL 与特征工程（Polars 向量化）。
4. `omega_v3_core/trainer.py`：v3 训练管线入口。
5. `omega_v3_core/README.md`：v3 入口与配置说明。

根目录代码（`kernel.py`/`omega_math_core.py`/`trainer.py`/`feature_extractor.py`/`data_adapter.py`）当前视为旧版或 L1 参考，不作为 v3 主要开发目标。

主要流程：
1. 数据 → Volume Clock → Returns → MDL（S/H）
2. Takens Embedding → TDA Filtration → Betti 指标
3. SRL Diagnostics → 残差/iceberg ratio
4. Regime 分类 + 方向性 → Signal
5. Entropy + SRL 残差 → Position Sizing
