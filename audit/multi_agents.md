# OMEGA Multi-Agents Design (Stable Path Spec)

## 1. Objective
- Provide a version-agnostic multi-agent collaboration layer on top of existing runtime pipelines.
- Keep data plane unchanged (`pipeline_runner.py`, `orchestrator.py`, `tools/v60_autopilot.py`, `tools/ai_incident_watchdog.py`).
- Keep file entrypoints stable across upgrades (`v6.1`, `v6.2`, ...).
- Follow guiding principles in `/Users/zephryj/work/Omega_vNext/audit/v60_multi_agents.md`.

## 2. Governance Baseline
- Human is final dispatcher and final merger.
- Git is the only code-state authority.
- No autonomous background merge between agents.
- One writer role at a time.
- Handoff protocol is Markdown-first.
- Every agent must read `/Users/zephryj/work/Omega_vNext/audit/constitution_v2.md` before any task.
- `/Users/zephryj/work/Omega_vNext/audit/constitution_v2.md` is immutable in normal task flow.

## 3. Role Model (Current Mainstack)
- Oracle / Planner / Orchestrator (`read_only`): `Codex 5.3` (`model_reasoning_effort=xhigh`)

## 7. Merge Gate
Before commit:
1. `04A_Gemini_Recursive_Audit.md` must output `PASS` or `REJECT`.
2. `04B_Codex_Recursive_Audit.md` must output `PASS` or `REJECT`.
3. Any `REJECT` blocks merge.
4. Human writes `05_Final_Audit_Decision.md` and performs manual final commit.

## 8. Recursive Audit Principle
- Each auditor must reason from first principles first, then implementation details.
- Each auditor must anchor findings to founding docs (`audit/constitution_v2.md`, `README.md`, `audit/multi_agents.md`, `.agent/principles.yaml`).
- Recursive depth minimum: 2 passes (direct defect -> downstream impact/regression risk).
- Auditors must not read each other's output before finishing their own.

## 9. Extension Contract
A new model/provider can be added only if:
1. It consumes one Markdown input and outputs one Markdown artifact.
2. It is bound to existing role protocol.
3. It does not bypass merge gate.
4. It is registered in `agent_profiles.yaml`.

## 10. Non-Goals
- No full autonomous multi-agent MLOps system.
- No replacement of worker orchestration.
- No hidden auto-commit/auto-merge recovery path.
