## 1. 训练配置 (Full Mode)

在 Debug 模式跑通后，我们移除了 `max_files` 限制，进行了全量数据的训练。

*   **数据源**: 
    *   `history_ticks_full`: 约 24,000 个高波动切片文件。
    *   `history_ticks`: 156 只股票的全年连续数据。
*   **校准采样**: `reservoir_size=50,000` 窗口，`max_windows_total=200,000`。
*   **训练轮次**: `epochs=1` (Pretrain) + `epochs=1` (Finetune)。
*   **Batch Size**: `512`。

## 2. Calibration 结果 (Pass-0)

全量数据的分布特征与 Debug 模式（小样本）有所不同，体现了更广泛的市场状态。

| 指标 | Debug (Low) | Debug (High) | **Full (Low)** | **Full (High)** | 洞察 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Epiplexity ($S$)** | 0.3546 | 0.4191 | **0.3866** | **0.4191** | 阈值区间收窄，说明全量数据的复杂度分布更集中，尾部特征更明显。 |
| **Entropy ($H$)** | 17.76 | 19.31 | **26.16** | **26.50** | **显著升高**。全量数据包含了大量高频噪声和高不确定性时段，模型需要适应更高的基准熵环境。 |

## 3. 模型特征重要性 (Feature Importance)

全量训练后的线性模型 (`SGDClassifier`) 权重揭示了真正具有普适性的 Alpha 来源。与 Debug 模式相比，Top 特征发生了显著变化：

| Rank | Feature | Weight | Physics Interpretation |
| :--- | :--- | :--- | :--- |
| 1 | **beta1_max** | **-0.2472** | **拓扑结构反转**: 这是一个强烈的负相关信号。$\beta_1$ 代表价格轨迹中的“持续循环”或“泡沫结构”。当拓扑结构最显著时（高 $\beta_1$），往往对应趋势的末端或即将崩塌的结构，因此预示下跌。这是 TDA 核心假设的有力验证。 |
| 2 | **H_bps** | **-0.2425** | **熵致衰减**: 在 Debug 模式中为正，但在全量数据中转为负。这表明在长周期下，极高的不确定性（高熵）往往伴随着价格的无序波动或下行风险（风险溢价消失），而非动能。 |
| 3 | **MDL_bps** | **-0.2384** | **复杂度惩罚**: 与熵类似，过高的描述长度意味着市场缺乏可利用的模式，模型倾向于在该区域做空或避险。 |
| 4 | **impact_pred** | **+0.1614** | **冲击动能**: 保持正相关。SRL 预测的冲击越大，买盘/卖盘的物理推动力越强，验证了流动性驱动的价格发现机制。 |
| 5 | **iceberg_ratio** | **+0.1296** | **流动性通畅度**: 正系数。`iceberg_ratio` = Realized / Predicted。当比率高（接近 1 或 >1）时，说明没有冰山单阻挡，价格能顺畅地响应冲击；当比率低时（有冰山），冲击被吸收，趋势难延续。 |

> **关键结论**: 全量模型学会了**“在结构崩塌前离场（Beta1），在物理动能强劲且无阻挡时进场（SRL + Iceberg）”**。

## 4. 回测结果 (Out-of-Sample)

我们在 `history_ticks` 的测试集（未参与训练的连续数据）上进行了回测。

*   **数据集**: 15 个文件（全年连续）。
*   **总交易数**: 1,209 笔。

| Metric | Value | 评价 |
| :--- | :--- | :--- |
| **Win Rate** | **46.07%** | 胜率略低于 50%，说明模型并非依靠高频胜率获利。 |
| **Total PnL** | **-0.0098** | 微亏，基本持平。 |
| **Sharpe Ratio** | **-0.0035** | 接近 0。考虑到包含交易成本，这说明原始 Alpha 勉强覆盖了成本，但缺乏超额收益。 |
| **Max Drawdown** | **-1.9187** | 回撤较大，存在某些极端行情下的连续亏损。 |

### 原因分析
1.  **线性局限性**: 尽管特征有效（权重显著），但线性模型（SGD）难以捕捉 Beta1 和 SRL 之间的复杂非线性交互（例如：只有在 Beta1 低且 Iceberg Ratio 高时才做多）。
2.  **交易频率低**: 1200 笔 / 15 股票 / 1 年 ≈ 80 笔/股/年。过低的频率导致样本方差极大，且错过了大量微观结构的短线机会。
3.  **成本敏感**: `cost_weight=1.0` 在低波动率环境下可能过重。

## 5. 改进路线图

基于全量回测结果，下一步的优化方向：

1.  **非线性模型**: 引入 `MLP` 或 `LightGBM` 替代 SGD，以学习特征间的条件概率（如“当熵低时，Beta1 的信号才有效”）。
2.  **增加频率**: 降低 `decision_margin` (当前 0.05) 或缩短 `lookback_bars`，尝试捕捉更高频的波动。
3.  **动态止盈止损**: 当前仅依赖固定时间窗口 (`label_horizon`) 的预测。引入基于 SRL 冲击预测的动态止盈（预测冲击耗尽即离场）可能显著提高胜率。
