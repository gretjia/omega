## 1. Context & Goal
The user requested a Stage 3 smoke test to validate the downstream pipeline (XGBoost model training and backtesting) using the V62 data currently outputted by the Linux node.
The requirement was to select approximately one week of data for training and another week for testing, ensure the pipeline runs successfully, and explicitly clean up any cache or temporary files to avoid polluting the final formal run.

## 2. Technical Decisions & Actions Taken
- Created a specialized local smoke test script (`tools/run_v62_smoke_test_linux.py`) that strictly uses the verified `.parquet.done` files from the `v62_feature_l2` directory.
- **Data Selection**:
  - Automatically scanned the chronologically sorted `.parquet.done` files on `linux1-lx`.
  - Selected 5 consecutive trading days (e.g., `20230103` to `20230112`) for **Train**.
  - Selected the next 5 consecutive trading days (e.g., `20230113` to `20230120`) for **Test/Backtest**.
- **Model Training**:
  - Loaded ~1.13 million training rows into memory.
  - Reused `OmegaTrainerV3._prepare_frames` to resolve T+1 predictive targets and process physical features.
  - Trained an `XGBClassifier` with `max_depth=6`, `learning_rate=0.1`, and `n_estimators=20`.
- **Model Backtesting**:
  - Saved the temporary model payload to disk.
  - Loaded ~1.12 million backtest rows from the separate test directory.
  - Ran the official `evaluate_frames` validation suite.
- **Strict Cleanup**:
  - Upon completion, the script explicitly deleted all symlinked directories (`/omega_pool/smoke_test_train`, `/omega_pool/smoke_test_test`) and the temporary `.pkl` model artifact to ensure a totally clean environment for the production run.
